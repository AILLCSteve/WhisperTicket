## Confirmed Architectural Facts

Non-obvious truths about how this code actually works — confirmed by reading source:

1. **"Hold-to-talk" is a toggle tap, not a hold gesture.** `LiveMicButton` is tap-to-start / tap-to-stop. No `LongPressGesture` exists. Calling it "hold-to-talk" in docs/comments is misleading.

2. **The 60-second ASR limit is bypassed by `requiresOnDeviceRecognition = true`.** The limit is server-side. Auto-restart in `SFSpeechTranscriptionService` is defensive for the server-fallback path (model not yet downloaded on first install), not a primary requirement.

3. **The audio session uses `.measurement` mode.** This disables system AGC and noise processing — correct for close-mic hold-to-talk but intentional. Don't change without understanding the tradeoff.

4. **`parseDraft()` is called on every partial ASR segment, not just on final results.** Runs at ~0.5s intervals during recording with no debounce. `FuzzyMenuOrderParser` does O(N×M) token overlap matching on every call.

5. **`TranscriptCleaner.clean()` is NOT called during live parsing.** It's called only in `finalizeTranscription()` as a fallback when zero items were parsed. The live parser uses its own 9-word stop list in `normalizeText()` — separate, smaller, no shared state.

6. **`reparseItems()` is merge-only.** Called when user edits a transcript. Never replaces existing items — only appends new ones. The same item can appear twice if the same text is processed twice with a new `DraftItem` UUID.

7. **Floor plan uses two separate canvas sizes.** Editor: 600×800 pt. Read-only embed: 900×700 pt. A table dragged to position (550, 750) in the editor would be clipped off the bottom of the embed. These have never been synchronized.

8. **`FloorPlanStore.load()` silently resets the floor plan to default on decode error.** `guard let plan = try? JSONDecoder().decode(...) else { return }` — returns without loading. Any schema change to `FloorPlanModels.swift` causes complete floor plan data loss with no user-visible error.

9. **`StubMenuImportService` is dead code.** `WhisperTicketApp.swift` line 59 wires `PDFMenuImportService`, not the stub. The stub documents the LLM Phase 2 blueprint.

10. **`TableSelectView.swift` is `typealias TableSelectView = FloorView`.** Superseded; safe to delete.

11. **`LocalBundleMenuStore` uses a 3-level load:** bundle JSON → UserDefaults → embedded fallback string constant. Not 5 levels as the old digest stated.

12. **No `deleteRule: .cascade` on ANY SwiftData relationship.** Default is `.nullify`. Every ticket delete leaves orphaned `GuestSeat`, `TicketItem`, `TicketModifier`, and `TicketEditEvent` rows in the store.

13. **`hasEntree` is almost always true from item 1.** `FuzzyMenuOrderParser` defaults `currentCourse = .entree`. The first item ordered (even a soda) gets `.entree`, triggering the dessert upsell immediately.

14. **PDF price regex captures integer only.** `(?:\.\d{2})` is non-capturing. `$14.99` → `14`. All imported menus have integer-dollar prices.

15. **PDF-imported items always have `tags: []`.** Tag-based upsell suggestions are completely inert for any menu produced by `PDFMenuImportService`.

16. **`@Observable` ViewModels are NOT marked `@MainActor`.** Mutations are protected at runtime by `.receive(on: .main)` in the transcription subscriber chain, but this is a convention, not a compiler guarantee. Any new async caller that forgets the hop is an undetected data race.

17. **`MARKETING_VERSION` in CI is hardcoded to "1.4.0" in `build.yml`.** The project.yml says "1.5.0 (8)". xcodebuild command-line args override project.yml. TestFlight shows 1.4.0.

## Recurring Failure Patterns

Root causes that appeared more than once in git history (from commits `307196a`, `33028e0`, `cc940b0`, `cd0df20`, `c0b4213`, etc.):

### 1. Smart/Curly Quotes in Swift String Literals (5+ occurrences)
- Commits: `33028e0`, `cd0df20`, `2067524`
- Root cause: AI training data includes rich-text markdown; Claude generates `"` / `"` naturally in quoted strings
- Symptom: `error: expected ',' separator` (misleading — not a separator issue)
- Fix: scan every new Swift file for U+201C/U+201D before committing
- **Still active risk on every Swift edit by Claude**

### 2. Transcription Race Conditions (2+ occurrences)
- Commits: `8f5e5d7`, `3f88ba9`, `477d156`
- Root cause: async boundaries between ASR result handler, audio engine, and ViewModel not always protected
- Symptom: partial transcripts, seat isolation failures, lost items
- Fix history: added `.receive(on: .main)`, fixed isFinal subscription, fixed per-seat isolation

### 3. CI Signing/Upload Failures (8+ commits)
- Commits: `fd7f012` through `a42c7a9` — extensive CI debugging
- Root cause: complex interplay of keychain setup, altool behavior, ASC API, JWT format
- Fixed patterns (see CLAUDE.md §CI rules): HTTP/2+curl, DER vs R‖S JWT, JSON:API fields[]

### 4. SwiftData Store Schema Mismatch (1+ occurrences)
- Commit: `470595c`
- Root cause: adding @Model properties without migration plan causes ModelContainer to throw
- Fix: store-wipe recovery in `WhisperTicketApp.init()` — intentional dev-only; must replace before production

### 5. Expression-Too-Complex in View Bodies (1 occurrence)
- Commit: `c63216d`
- Root cause: `TicketEditorView.body` exceeded Swift type-checker budget
- Fix: decompose into `@ViewBuilder` private functions; watch `TableOrderEntryView` as next candidate

## Active Watch-Outs

Current conditions in the code that cause non-obvious failures. All verified still present as of 2026-05-19.

### [CRITICAL] No AVAudioSession Interruption Handler
- **File**: `AudioCaptureService.swift`
- **What**: No observer for `AVAudioSession.interruptionNotification`
- **Trigger**: Phone call arrives mid-recording
- **Failure**: System deactivates session, engine loses input, `isRecording = true` but engine is stopped. App is in inconsistent state. Next recording attempt may fail silently.
- **Fix**: Register observer in `startCapture()`. On `.began`: call `stopCapture()`, notify ViewModel via publisher. On `.ended` with `.shouldResume`: re-enable session only if user re-initiates.

### [CRITICAL] PDF Price Regex Captures Integer Only
- **File**: `PDFMenuImportService.swift:44`
- **What**: `\$?\s*(\d{1,3}(?:\.\d{2}))` — `(?:...)` is non-capturing. Capture group 1 captures `\d{1,3}` only.
- **Trigger**: Any PDF menu import
- **Failure**: $14.99 → price 14.0. Every imported menu has integer-dollar prices. Silent data corruption.
- **Fix**: Change to `(\d{1,3}\.\d{2})` (remove `?:`).

### [CRITICAL] No Cascade Delete on SwiftData Relationships
- **Files**: `Models/Ticket.swift`, `Models/GuestSeat.swift`, `Models/TicketItem.swift`
- **What**: Default SwiftData delete rule is `.nullify`. No `@Relationship(deleteRule: .cascade)` declared.
- **Trigger**: Any ticket deletion
- **Failure**: Deleting a `Ticket` leaves all its `GuestSeat`, `TicketItem`, `TicketModifier`, and `TicketEditEvent` orphaned in the store. Accumulates indefinitely.
- **Fix**: Add `@Relationship(deleteRule: .cascade)` to all four relationship levels. Note: cascade + explicit `save()` in same transaction is buggy (FB13640004) — delete children manually before parent if using explicit saves.

### [HIGH] @Observable ViewModels Missing @MainActor
- **Files**: `ViewModels/TicketEditorViewModel.swift`, `ViewModels/LiveSessionViewModel.swift`
- **What**: Classes are `@Observable` but not `@MainActor`. Mutations are protected at runtime by `.receive(on: .main)` in the subscription chain, but any new async caller that forgets the hop introduces a silent data race.
- **Trigger**: Any new async context that mutates ViewModel properties without explicit main-actor hop
- **Fix**: Add `@MainActor` to all `@Observable` ViewModels. This is a compile-time guarantee with zero runtime behavior change given the existing `.receive(on: .main)` already in place.

### [HIGH] Buffer Not Copied Before Combine Send
- **File**: `AudioCaptureService.swift` — `installTap` callback
- **What**: `bufferSubject.send(buffer)` called from RT audio thread without copying buffer data. Engine can reuse internal buffer memory before the downstream subscriber processes it.
- **Trigger**: High load, buffer reuse timing
- **Failure**: Latent crash — rare but non-deterministic
- **Fix**: In the `installTap` callback, create a new `AVAudioPCMBuffer`, copy the data, then send the copy.

### [HIGH] FloorPlanStore Silent-Reset on Decode Error
- **File**: `Services/FloorPlanStore.swift` — `load()` method
- **What**: `guard let plan = try? JSONDecoder().decode(...) else { return }` — decode failure returns without loading, leaving the store in default state.
- **Trigger**: Any schema change to `FloorPlanModels.swift` (adding non-optional field, renaming, etc.)
- **Failure**: Complete floor plan data loss with no error shown to user.
- **Fix**: At minimum log the decode error. Ideally: on failure, preserve the raw bytes in a separate key, show a recovery alert, and don't silently reset.

### [MEDIUM] @Query Without fetchLimit in TicketsListView
- **File**: `Views/TicketsListView.swift` (uses `@Query`)
- **What**: `@Query` macro does not support `fetchLimit`. All historical tickets are loaded into memory.
- **Trigger**: App usage over time with many completed tickets
- **Failure**: Unbounded memory growth; performance degradation with 100+ tickets
- **Fix**: Replace `@Query` with `FetchDescriptor` + `modelContext.fetch()` with `fetchLimit` and predicate filtering (e.g., only open/recent tickets by default).

### [MEDIUM] parseDraft() No Debounce
- **File**: `ViewModels/LiveSessionViewModel.swift` — `handleTranscriptionSegment()`
- **What**: `parser.parseDraft()` called on every partial ASR result (~every 0.5s) with no debounce.
- **Trigger**: Any recording session
- **Failure**: O(N×M) fuzzy matching 2× per second. On large menus, noticeable CPU usage.
- **Fix**: Add `.debounce(for: .milliseconds(300), scheduler: DispatchQueue.main)` before `handleTranscriptionSegment`. Bypass for `isFinal == true` results.

### [MEDIUM] Modifier Negation Bug
- **File**: `Services/FuzzyMenuOrderParser.swift` — `extractModifiers()`
- **What**: Negation creates a `TicketModifier` with name `"No Ranch"` (string prefix) instead of using the existing `isNegation: Bool` field.
- **Trigger**: Any "no X" or "hold the X" modifier in speech
- **Failure**: `TicketModifier.isNegation` is never set to `true`. Negation display and kitchen note logic that reads `isNegation` never fires.
- **Fix**: Create `TicketModifier(name: "Ranch", isNegation: true)` instead of `TicketModifier(name: "No Ranch")`.

### [MEDIUM] hasEntree Default-Course Bug
- **File**: `Services/FuzzyMenuOrderParser.swift`
- **What**: `currentCourse` defaults to `.entree` for all items unless an explicit course keyword is spoken. All items (including beverages, sides) get `.entree` by default.
- **Trigger**: Any order where course keyword is not spoken
- **Failure**: `RuleBasedUpsellEngine.hasEntree` is true from item 1. Dessert and beverage upsells fire immediately on the first ordered item.
- **Fix**: When a matched menu item has tags, infer `CourseFlag` from them (priority: spoken keyword > tag inference > default).

### [MEDIUM] Canvas Size Mismatch (Floor Plan)
- **Files**: `Views/FloorPlanEditorView.swift` (600×800), `Views/FloorView.swift` → `FloorMapEmbedView` (900×700)
- **What**: Editor and operational embed have different fixed canvas sizes.
- **Trigger**: Tables dragged to the 600-800 range in the editor appear clipped or off-canvas in the embed.
- **Fix**: Unify to 900×700 in both. Or make canvas size a `FloorPlan` property so it's consistent.

### [LOW] allowBluetooth Deprecation (Xcode 26)
- **File**: `Services/AudioCaptureService.swift`
- **What**: `.allowBluetooth` deprecated in Xcode 26. Same raw value as `.allowBluetoothHFP` at runtime — no behavioral impact on current iOS.
- **Fix**: Change `.allowBluetooth` → `.allowBluetoothHFP` on next `AudioCaptureService` edit. Use `#if compiler(>=6.2)` shim if supporting multiple Xcode versions.

### [LOW] MARKETING_VERSION CI Mismatch
- **File**: `.github/workflows/build.yml`
- **What**: Archive step hardcodes `MARKETING_VERSION="1.4.0"`. project.yml says `1.5.0 (8)`. CI wins.
- **Fix**: Remove the hardcoded override from `build.yml`. Let project.yml be the source of truth.

### [LOW] No VersionedSchema Before Production
- **Files**: All 5 `@Model` types
- **What**: Unversioned schema. Once shipped to real users, adding a `SchemaMigrationPlan` later throws "Cannot use staged migration with an unknown model version" — cannot be retrofitted cleanly.
- **Fix**: Add `SchemaV1` + empty `WaitTicketMigrationPlan` before first production release. Zero stages = zero migration cost now; pays off on first property addition.

## Claude-Specific Codegen Mistakes

What Claude specifically gets wrong on this stack — verified from git history and session experience:

| Mistake | Symptom | Prevention |
|---|---|---|
| **Smart/curly quotes in Swift string literals** | `error: expected ',' separator` | Scan every new Swift file for U+201C/U+201D before committing |
| **Missing `@MainActor` on `@Observable` classes** | Silent data races, rare crashes | Always annotate `@Observable` ViewModels with `@MainActor` |
| **`vm.` scope confusion** | Type error — property is on View (`@State`), not ViewModel | Transient UI state (sheet open/close, editing selection) belongs on View as `@State`, not ViewModel |
| **Expression-too-complex in large View bodies** | `error: the compiler is unable to type-check this expression` | Decompose into `@ViewBuilder` private functions after ~4 sections. Proactively, not reactively. |
| **Deprecated API usage** | Compiler warnings that accumulate | Fix `.allowBluetooth`→`.allowBluetoothHFP` on next AudioCaptureService edit; check for others on each file touch |
| **Non-capturing `(?:...)` in regex capture groups** | Silent logic bug (extracted wrong value) | Verify capture group behavior; `(?:...)` does NOT capture |
| **Hardcoded "drink" instead of project-canonical "beverage"** | Tag mismatch, upsell silent failure | Check MenuV1.sample.json tag taxonomy before writing any tag string |
| **`.onChange(of: vm)` watching whole ViewModel** | Closure never fires | Always `.onChange(of: vm.specificProperty)` |
| **In-place array mutations on `@Observable`** | Full view reinitialization instead of targeted update | Use copy-then-replace for all array mutations on `@Observable` objects |
