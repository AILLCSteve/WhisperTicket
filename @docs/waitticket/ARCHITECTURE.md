## Key Patterns

### Dependency Injection
`AppServices` struct holds all service instances. Injected at root via:
```swift
@Entry var appServices = AppServices()
```
Consumed in views via `@Environment(\.appServices) var services`. Swap concrete types in `WhisperTicketApp.swift` — the rest of the app is unaffected because all services conform to protocols.

**Rule:** All new services must conform to a protocol in `Protocols.swift` and be wired in `WhisperTicketApp.swift`. Never instantiate services in views.

### State Management
- `@Observable` classes for ViewModels — NOT `ObservableObject`
- Observation is **per-property** (not whole-object): reading `vm.items` does not subscribe to `vm.isRecording`
- **Thread safety**: `@Observable` does NOT enforce `@MainActor`. All ViewModels that back SwiftUI views must be annotated `@MainActor` (currently missing — see Active Watch-Outs)
- `.onChange(of: vm)` trap: watches reference identity, never fires on property mutations — always use `.onChange(of: vm.someProperty)`
- Array mutations on `@Observable` objects need copy-then-replace (`var copy = vm.arr; copy.append(x); vm.arr = copy`) or SwiftUI sees a full object replacement

### Persistence
**SwiftData (tickets):**
- `ModelContainer` created in `WhisperTicketApp.init()` with store-wipe recovery on init failure
- Store-wipe recovery: catches `ModelContainer` throw, deletes `.sqlite`/`.sqlite-wal`/`.sqlite-shm`, retries — intentional dev-only pattern
- No `VersionedSchema` or `MigrationPlan` in place — must add before production
- Default delete rule is `.nullify` — no cascade configured on any relationship (orphans accumulate on ticket delete)
- `@Query` in list views has no `fetchLimit` — loads all tickets into memory

**UserDefaults (floor plan):**
- `FloorPlanStore.load()` uses `guard let plan = try? JSONDecoder().decode(...) else { return }` — **silent reset** on any decode error (schema change wipes the floor plan without warning)
- `upsertTable()` calls `save()` synchronously on every drag-end (no debounce)

### Audio Buffer Pipeline
```
AVAudioEngine.inputNode.installTap (RT audio thread)
  → PassthroughSubject<AVAudioPCMBuffer, Never>.send()   ← RT thread
  → audioCancellable.sink { request?.append(buffer) }    ← still RT thread
  → SFSpeechRecognitionTask result handler               ← Speech framework thread
  → transcriptionSubject.send(segment)                   ← Speech framework thread
  → .receive(on: DispatchQueue.main)                     ← hops to main (CORRECT)
  → handleTranscriptionSegment()                         ← main thread
```

**Critical**: buffer is NOT copied before `PassthroughSubject.send()` — engine can reuse internal buffer memory before subscriber processes it (latent crash). Must copy buffer in tap block before sending.

### Protocol Abstraction (Phase 2 readiness)
Every service has a protocol in `Protocols.swift`. The concrete swap happens only in `WhisperTicketApp.swift`. This means:
- `WhisperKitTranscriptionService` replacing `SFSpeechTranscriptionService` = one line change
- `SupabaseTicketRepository` replacing `SwiftDataTicketRepository` = one line change
- `OpenAIMenuImportService` replacing `PDFMenuImportService` = one line change

## Library-Specific Notes

### @Observable (iOS 17+)
- Observation granularity: per accessed key path, not whole object. Pass full VMs to child views — the framework tracks which properties each view actually reads.
- `@State` with `@Observable` init runs on every rebuild — only declare VMs as `@State` at App level or stable parent views. Leaf views must receive VMs via parameter or `@Environment`.
- `@ObservationIgnored`: mark high-frequency non-UI properties (AVAudioEngine ref, audio buffers, internal counters) to prevent spurious view updates.
- `@Bindable`: correct way to get `Binding<T>` projections from `@Observable` objects received via parameter or environment (not `@State`).
- Collections in `@Observable`: always copy-then-replace. Direct in-place mutations (`arr.append(x)`) can cause full view reinitialization.

### SwiftData (iOS 17)
- `ModelContainer` throws `NSPersistentStoreIncompatibleVersionHashError` (134504) on schema mismatch with existing store. Store-wipe recovery in `WhisperTicketApp.init()` handles this.
- Cascade delete: default rule is `.nullify`. Must declare `@Relationship(deleteRule: .cascade)` explicitly on all parent relationships. **Known bug (FB13640004)**: cascade + explicit `save()` = orphans. Delete children manually before parent when using explicit saves.
- `@Model` objects are non-`Sendable` — cannot cross actor boundaries. Pass `PersistentIdentifier` across actors, use `@ModelActor` for background processing.
- `@Query` has no `fetchLimit`, `propertiesToFetch`, or `relationshipKeyPathsForPrefetching` — use `FetchDescriptor` + `modelContext.fetch()` for list views.
- Lazy relationship loading: `ticket.seats` triggers a fetch; `seat.items` triggers another. Use `FetchDescriptor.relationshipKeyPathsForPrefetching = [\.seats]` to batch.
- Insert order matters: always insert parent before children or save may throw.
- `deleteAllData()` silently no-ops on iOS 17.0 — delete `.sqlite` files directly (current approach is correct).

### AVAudioEngine + AVAudioSession
- `installTap` callback fires on real-time audio thread — no allocation, no locking, no actor isolation. Call `request?.append(buffer)` synchronously; copy buffer data before any async handoff.
- `.measurement` mode (used in project): disables system AGC and noise processing — correct for close-mic hold-to-talk; may hurt accuracy in high-noise if system suppression is desired.
- `.allowBluetooth` → `.allowBluetoothHFP`: same raw value at runtime, but `.allowBluetooth` deprecated in Xcode 26. Fix on next `AudioCaptureService` edit.
- No `AVAudioSession.interruptionNotification` observer — phone call mid-recording leaves `isRecording = true` with the engine stopped. **Must add before production.**
- `AVAudioSession.setActive(true)` is synchronous and can fail silently if interrupted and not re-activated after.

### SFSpeechRecognizer
- `requiresOnDeviceRecognition = true` bypasses the 60-second server-side limit — auto-restart logic in `SFSpeechTranscriptionService` is defensive coding for the server-fallback case (model not yet downloaded on first install).
- `isAvailable` returns `true` on iOS 17 even when dictation is disabled — tasks fail immediately. Check `supportsOnDeviceRecognition` at launch.
- `SFCustomLanguageModelData` (iOS 17+): builds a custom on-device language model from `PhraseCount` templates and `CustomPronunciation` entries. Set `recognitionRequest.customizedLanguageModel = url`. This project does not use it — adding menu item names as templates would meaningfully improve accuracy. Requires `requiresOnDeviceRecognition = true` (already set).
- Recognition task result handler fires on a Speech framework internal thread — always hop to main before `@Observable` mutations.
- Explicit `task.cancel()` required before nil-ing the task — setting `recognitionTask = nil` without cancel leaves the XPC task running and leaks the result handler block.
- Teardown order: cancel task → removeTap → stop engine → endAudio (clean path only) → nil refs → deactivate session.

### Combine (audio pipeline)
- `PassthroughSubject.send()` dispatches synchronously to all subscribers on the calling thread. If called from RT audio thread, all subscribers run on RT thread unless `.receive(on:)` is applied.
- Buffer lifetime: `AVAudioPCMBuffer` from installTap is valid only within the callback. Copy before sending through Combine.

## Build & CI

### xcodegen
- `project.yml` is the source of truth for project structure. Run `xcodegen generate` before any `xcodebuild` command in CI.
- `MARKETING_VERSION: "1.5.0"` in project.yml, but `build.yml` archive step hardcodes `MARKETING_VERSION="1.4.0"` — CI value wins. Ships 1.4.0. Fix: remove the hardcode from `build.yml` and let xcodegen's value be read by xcodebuild.
- Info.plist uses `$(MARKETING_VERSION)` — reading it in shell scripts returns the literal string. Use `xcodebuild -showBuildSettings | grep MARKETING_VERSION | awk '{print $3}'` to read the actual value.

### GitHub Actions
- `macos-latest` = macos-15 (Xcode 16) as of August 30, 2025. Pin to `macos-14` for Xcode 15 stability.
- `security authorizationdb write com.apple.trust-settings.admin allow` fails with error -60005 on macOS 15. Remove from signing scripts.
- Manual code signing: 6-step keychain pattern required (create → set timeout → unlock → import cert → set-key-partition-list → install profile). Missing `set-key-partition-list` causes silent hang.
- `security set-keychain-settings -lut 21600` (6-hour timeout) required for long builds.

### xcrun altool
- `--upload-app`: functional on Xcode 14–16; buggy on Xcode 26 (Fastlane issue #29743).
- `--notarize-app`: fully deprecated since Nov 2023 (macOS-only, not relevant for iOS).
- ASC JWT: Python `cryptography` library + PyJWT for ES256 signing. bash+openssl produces DER format, not the R‖S format ASC requires. Current approach is correct.
- JWT lifetime: max 1200 seconds (20 min). Generate JWT immediately before the upload call — not at workflow start.

### Free Apple ID limitation
- Cannot generate App Store Connect API keys (requires paid Developer Program account).
- Cannot upload to TestFlight. The entire CI upload path is blocked without a paid account.
- `xcrun altool` upload will always fail with authentication error on a free account.
